程序目的：应用Intel SSE指令集和多线程编程，实现图像融合，最终的效果是电子相册的形式，淡入淡出
遇到的问题：
	1.SSE指令集不熟悉，不会用SSE2中增加的指令对整型向量进行处理，不懂具体如何进行pack和unpack；
	解决办法：参考官方手册和网上博客；
	MMX/SSE/AVX使用手册
	https://software.intel.com/sites/landingpage/IntrinsicsGuide/#expand=954,950,3108,950,4872,4873,4874,4875,4881,659,2895,3660,159,79,2395,15,25,24,5331,124,5554&othertechs=XSS&techs=SSE
	基于SSE4和多核编程的电子相册的实现
	https://blog.csdn.net/lyj2018gyq/article/details/82057108
	2.多线程编程出现崩溃
	现象a:开启两个线程分别对两个文件夹下图片进行融合并显示，启动后只有一个窗口且来回显示来自不同文件夹的图片，大约1秒后崩溃退出；
	原因a:采用了全局静态变量对线程计数，并在线程函数内部用此数字命名了窗口，起初是想按照线程的编号命名窗口，但是由于并行的原因，绝大部分时候两次窗口命名用的数字都是“2”，导致冲突；
	解决a:直接给线程函数传入参数进行窗口命名，保证名称不冲突；

	现象b:debug模式下，程序每次都在imageImport()这个函数执行期间弹出“debug error”，不论点击“终止、调试、忽略”，程序都会退出，但不动的话可以正常显示；
	原因b:开辟两个线程后，没有用thread.join()，在子线程执行期间，主线程已经结束退出了。
